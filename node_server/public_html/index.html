The REST API

The server acts as a simple blog where users can add or retrieve all current "blog" posts. The API for doing this is described below. The server manages sessions using cookies, so when the Worklight server has received a response (when trying to log in), the cookie contained in the response should be used in further requests. An example request could be http://192.168.2.39:8080/posts.

Getting blog posts

method: GET

url: /posts

parameters: none

response type: JSON

response: { posts: [{

title: 'Example'

author: 'Anton'

content: 'This is an example'

}] }

Logging in

method: POST

url: /session

parameters: "username", "password"

Adding blog posts

method: PUT

url: /posts

parameters: "content", "title"


Logging out

method: DELETE

url: /session

parameters: none


Registering a user

method: PUT

url: /users

parameters: "username", "password"
